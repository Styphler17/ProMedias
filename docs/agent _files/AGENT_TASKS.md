# Agent Tasks — Claude Execution Checklist

## Phase 1 — Discovery
- Read every file in `docs/`
- Create a concise fact sheet from project documents
- Extract categories, services, policies, contact info, branding, and constraints
- Flag missing details

## Phase 2 — Strategy
- Recommend best stack for this client
- Decide whether WordPress/WooCommerce is better than headless, based on maintenance and docs
- Produce sitemap and URL structure
- Define multilingual approach

## Phase 3 — Data modeling
- Create JSON or CMS schemas for products, categories, services, FAQs, and settings
- If spreadsheets exist, map columns into the chosen schema
- Normalize brand/category names

## Phase 4 — UX/UI system
- Define colors, fonts, spacing, buttons, cards, badges, forms, nav, footer
- Create desktop and mobile wireframe notes
- Specify reusable components before coding pages

## Phase 5 — Build
- Build shell/layout first
- Then homepage
- Then category and product templates
- Then repair pages
- Then cart/checkout
- Then policy and contact pages

## Phase 6 — SEO and trust
- Add schema markup
- Add local SEO blocks
- Add crawlable intro text to categories
- Add metadata strategy
- Add Open Graph defaults

## Phase 7 — QA
- Test mobile first
- Validate forms and CTAs
- Check image handling and lazy loading
- Check accessibility basics
- Check page speed basics

## Non-negotiables
- Must use docs folder contents
- Must avoid fake content when real source material exists
- Must annotate assumptions clearly
- Must keep code modular and maintainable
