# Deployment Guide — LWS.fr (React + PHP MVC + MySQL/MariaDB)

This guide provides step-by-step instructions for deploying the **ProMedias** storefront to LWS shared hosting using the PHP MVC backend.

---

## 🏗️ Architecture Overview

The app is divided into three main parts:
1.  **Frontend**: Static assets build from React (Vite).
2.  **Backend (MVC)**: Professional PHP 8.x Bridge (`api/` folder) handling routing and Database.
3.  **Database**: MySQL/MariaDB instance managed from the LWS Panel.

**Recommended Plan**: LWS Linux shared hosting with PHP 8.x and MySQL. Node.js is not required on the server.

---

## 🛠️ Step 1 — Database Setup

1.  Log in to **LWS Panel**: `https://panel.lws.fr`.
2.  Open the hosting service with **Gérer**.
3.  Go to **Base de données & PHP → MySQL & PHPMyAdmin**.
4.  Create a MySQL database.
    *   **MySQL Database**: use the name generated by LWS, for example `promedias_xxxxx`.
    *   **MySQL Username**: use the user generated by LWS.
    *   **Password**: create or copy the database password from LWS Panel.
    *   **Host**: use the database host shown by LWS. It is often `localhost`, but do not assume it if LWS displays another hostname.
5.  Open **phpMyAdmin** for this database.
6.  Click **Import** and upload the latest SQL export, usually `docs/db/promedias.sql` or an up-to-date export from local phpMyAdmin.
    *   Ensure the `admin_users` table has at least one account before testing `/admin`.

LWS references:
- Create a database: https://aide.lws.fr/base/Hebergement-web-mutualise/Bases-de-donnees-Mysql/Comment-creer-une-base-de-donnees-Mysql
- Access phpMyAdmin: https://aide.lws.fr/base/Hebergement-web-mutualise/Bases-de-donnees-Mysql/bases-de-donnees-Mysql-PhpMyAdmin
- Import with phpMyAdmin: https://aide.lws.fr/base/Hebergement-web-mutualise/Bases-de-donnees-Mysql/importer-base-de-donnees-php-my-admin-lws-panel

## 🚀 Step 2 — Preparing the Build

1.  Run the build command locally:
    ```bash
    npm run build
    ```
    *This generates a `dist/` folder.*
2.  Keep `VITE_CMS_URL=https://promedias-liege.be` when the `api/` folder is deployed under the same web root. If you deploy the API on a separate subdomain, set `VITE_CMS_URL` to that subdomain origin, without a trailing `/api`.

---

## 📦 Step 3 — Uploading to LWS

Using **LWS File Manager** or FTP/SFTP, upload your files to the domain web root. Depending on the LWS product, this may be named `htdocs`, `www`, or the domain's configured document root. The final structure must look like this:

```text
web-root/
├── api/                <-- Copy the ENTIRE api/ folder from project root
│   ├── Controllers/
│   ├── Core/
│   ├── Models/
│   ├── .htaccess
│   ├── config.php      <-- EDIT THIS ON SERVER
│   └── index.php
├── uploads/            <-- Copy your local uploads/ folder
├── index.html          <-- From your local dist/ folder
├── assets/             <-- From your local dist/ folder
└── .htaccess           <-- Create this for React routing
```

## ⚙️ Step 4 — Final Configuration

### 1. Update Database Credentials
Open **`api/config.php`** on the server and update the `db_config` array:

```php
$db_config = [
    'host' => 'localhost',              // Or the DB hostname shown in LWS Panel
    'port' => '3306',
    'user' => 'your_lws_db_user',
    'pass' => 'your_lws_db_password',
    'name' => 'your_lws_db_name'
];
```

Alternative: keep `api/config.php` generic and create `api/.env.php` on the server:

```php
<?php
$db_config = [
    'host' => 'localhost',
    'port' => '3306',
    'user' => 'your_lws_db_user',
    'pass' => 'your_lws_db_password',
    'name' => 'your_lws_db_name'
];
```

### 2. Configure Root `.htaccess` (React Routing)
The root `.htaccess` from `public/.htaccess` is copied into `dist/` during build. Confirm it exists in the **web root** after upload, not only inside `api/`.

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```

### 3. Permissions
Ensure the **`uploads/`** folder has **755** permissions so the backend can read/write images if needed.

### 4. PHP Version
In LWS Panel, configure the site to use PHP 8.x, preferably PHP 8.2 or newer. If you are on LWS cPanel, PHP versions are selected from cPanel's PHP selector.

LWS references:
- Configure PHP on LWS shared hosting: https://aide.lws.fr/base/Hebergement-web-mutualise/Outils-web/Configurer-PHP
- Change PHP version on LWS cPanel: https://aide.lws.fr/base/cPanel/Configuration-PHP-avec-cPanel/Comment-changer-de-version-PHP-sur-votre-hebergement-cPanel

## ✅ Deployment Checklist

- [ ] MySQL tables imported through LWS phpMyAdmin.
- [ ] PHP version set to PHP 8.x.
- [ ] `api/config.php` or `api/.env.php` updated with LWS database credentials.
- [ ] `dist/` contents uploaded to root.
- [ ] `api/` folder uploaded to root.
- [ ] `uploads/` folder uploaded to root.
- [ ] Root `.htaccess` uploaded for React Router refresh support.
- [ ] `api/.htaccess` uploaded for API route rewriting.
- [ ] `uploads/` is writable.
- [ ] Test the API: Visit `yoursite.com/api/ping` (Should return JSON).
- [ ] Test data endpoints: Visit `yoursite.com/api/products?status=published`.
- [ ] Test the App: Visit `yoursite.com` and verify products load.
- [ ] Test Admin: Visit `yoursite.com/admin` and try to login.
